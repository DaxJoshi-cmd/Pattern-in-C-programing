Introduction

In C programming, pattern problems involve printing a specific arrangement of characters, numbers, or symbols.

These patterns are generated using loops, especially nested loops, to control rows and columns.

What is a Loop?

A loop is a control structure that allows a set of instructions to be executed repeatedly until a given condition is satisfied.

Benefits (from GeeksforGeeks):

  • Reduces code repetition (DRY principle)
  • Improves program efficiency
  • Essential for handling repetitive tasks like pattern printing

Warning: Always ensure the loop condition will eventually become false to avoid infinite loops!

Types of Loops in C (GeeksforGeeks)

  • for loop ‚Üí Entry-controlled. Best when number of iterations is known.
  • while loop ‚Üí Entry-controlled. Condition checked before execution.
  • do-while loop ‚Üí Exit-controlled. Executes at least once.

Key Difference: while/for check before; do-while checks after.

Why Nested Loops for Patterns?

Loop Optimization Techniques in C

  • Loop Unrolling: Repeat body manually
  • Minimize Inner Work: Compute constants outside
  • Use ++i over i++
  • Compiler Flags: -O2 / -O3

Pascal's Triangle ‚Äì Code

int n = 6;
for(int i = 0; i < n; i++) {
    for(int s = 0; s < n-i; s++) printf("  ");
    int val = 1;
    for(int j = 0; j <= i; j++) {
        printf("%4d", val);
        val = val * (i - j) / (j + 1);
    }
    printf("\n");
}

Diamond Pattern ‚Äì Code

int n = 5;

// Upper half
for(int i = 1; i <= n; i++) {
    for(int s = 1; s <= n-i; s++) printf(" ");
    for(int j = 1; j <= 2*i-1; j++) printf("*");
    printf("\n");
}

// Lower half
for(int i = n-1; i >= 1; i--) {
    for(int s = 1; s <= n-i; s++) printf(" ");
    for(int j = 1; j <= 2*i-1; j++) printf("*");
    printf("\n");
}

Common Mistakes & Debugging Tips

  • Off-by-One Error: Loop runs one time too many/few
  • Forgetting \n: All output appears on one line
  • Wrong space calculation: Misaligned patterns
  • Infinite loop: Condition never becomes false
  • Debugging Tips:
    • Use temporary printf to see row/col values
    • Dry-run on paper for small n
    • Test with n=1 and n=2 first
